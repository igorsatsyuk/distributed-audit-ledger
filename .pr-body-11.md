## Changes

### `audit-log.model.ts`
- `BlockchainRecord` — интерфейс для blockchain-записи (`exists`, `transactionHash`, `blockNumber`, `timestamp`)
- `IntegrityCheckResponse` — полный ответ `/integrity-check`
- `AuditLogFilters` — добавлены `from`, `to`, `limit`, `offset`

### `audit-log.service.ts`
- `getAuditLogById(id)` → `GET /api/audit-logs/{id}`
- `checkIntegrity(id)` → `GET /api/audit-logs/{id}/integrity-check`
- `getAuditLogs` передаёт все фильтры включая пагинационные параметры

### `audit-dashboard.component.ts`
- Сигналы: `pageSize`, `pageIndex`, `estimatedTotal`, `integrityLoading`, `integrityCheckResult`, `integrityCheckError`
- При `openDetails()` автоматически вызывается `checkIntegrity` с live-результатом в drawer
- `onPageChange(event: PageEvent)` для пагинатора
- `retry()` для перезапроса при ошибке
- `ngOnDestroy` + `takeUntil(destroy$)` — отмена in-flight запросов при уничтожении компонента

### `audit-dashboard.component.html`
- `mat-progress-bar` на уровне страницы при загрузке
- Error-block с иконкой `error_outline` и кнопкой Retry
- `mat-paginator` с `showFirstLastButtons`
- Drawer: секция Blockchain verification — loading-bar, статус-чип, `dl`-список с txHash / blockNumber / blockTime
- Строка «No audit events found» (`*matNoDataRow`)

### `audit-dashboard.component.scss`
- Стили: `.error-block`, `.detail-list`, `.detail-list--blockchain`, `.mono`, `.hash-value`, `.page-progress`
